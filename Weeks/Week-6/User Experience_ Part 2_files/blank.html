Portal technical architecture and widgets

Lesson 6 of 8

Portal framework

The portal framework is composed of a set of APIs, Angular services, directives, and tools that help to build portals. A portal is a group of pages linked by page IDs.

Select the (+) icon to learn more about the framework components.

Note: The default portal in the base system is now /esc.

To invite a user to a portal, you provide them with a URL (e.g. https://yourdomain.service-now.com/esc).

Widgets and the Widget Editor

Widgets display content on pages and perform one function. They are reusable components that can be individually customized to:

  • Present data

  • Perform an action

  • Access a service

The same widget can be added multiple times on a page and can have a different configuration every time it is used. More than 190 widgets are provided in the base system. They are:

  • Self-contained

  • Secure

  • Practical - search, requests, forms, lists, connect, and more!

With the Widget Editor, you can create widgets from scratch or customize an existing one. You can write HTML, CSS, and Javascript with real-time preview. It is a leading practice to use the Widget Editor to develop and modify Service Portal Widgets. It lets you choose which parts of a widget you want to edit and hides the rest.

Keep in mind

  • Widgets are extensible and re-usable. When developing, ask yourself, "Can this be used on another page?”
  • Create widgets with reuse in mind, thus making future page development faster


Optional: Check out the Widget Developer Guide(opens in a new tab) for more information.

Note: Before creating a custom widget, check the Widget Library(opens in a new tab) if one already exists.

Widget behavior

Widgets execute a specified behavior on a page. Select each tab to learn more.

  • Render the view, what end users see in their browser
  • Use information from the model and controller
  • Bind client script variables to your markup
  • Gather data from the end user
  • Load the initial data in the widget
  • Send record data to the widget client script (Controller)
  • Run server-side queries/scripts
  • Map server data from JavaScript and JSON objects to client objects
  • Process data before rendering it
  • Pass user input and data back to the server for processing

Style a widget

External CSS can also be brought in by copying and pasting the code into a style sheet record and then attaching it to a theme record using the CSS Includes related list. Once created, a style sheet can be attached to any theme record.

Use the CSS – SCSS editor to create specific styles for the widget.

Bootstrap and Font Awesome

Bootstrap and Font Awesome are installed in the platform and can be referenced in HTML.

Bootstrap offers reusable components built to provide iconography, dropdowns, input groups, navigation, alerts, and much more.

Font Awesome provides an icon library + toolkit. Service Portal developers do their best to include the most current release of Font Awesome with every new ServiceNow release.

Hierarchy of applying style

It is important to understand the hierarchy of the styles applied in your Service Portal and what CSS will be overwritten by which component.
